# DrawClaw - Expert Visual Reasoning Engine

## Philosophy

Drawing is not copying.

Drawing is structured problem solving using observation, design judgment, and intentional decision-making.

This reference system teaches the agent to think like a senior artist, not like a tutorial search engine.

## Folder Philosophy

Every folder serves one of three purposes:

| Purpose | Description | Example |
|---------|-------------|---------|
| **Foundation** | Core skills that enable all others | gesture_drawing, form_breakdown, perspective |
| **Application** | Real-world production usage | concept_art_pipeline, visual_storytelling |
| **Judgment** | Decision-making and critique | professional_critiques, design_tradeoffs |

## Priority Hierarchy

When answering a drawing question, the agent should consult folders in this order:

1. **critique/** — Identify the problem, not just the solution
2. **foundations/** — Address root causes, not symptoms
3. **anatomy_advanced/** — Apply structural knowledge
4. **production/** — Consider real-world workflow
5. **mental_models/** — Guide thinking, not just technique

## How the Agent Should Reason

### For Technical Questions ("How do I draw hands?")
1. Start with **foundations** (gesture, form, construction)
2. Layer in **anatomy_advanced** (hands_feet)
3. Add **critique** (common mistakes)
4. End with **mental_models** (how to practice effectively)

### For Creative Questions ("How do I design a character?")
1. Start with **mental_models** (decision-making framework)
2. Use **foundations** (shape_language)
3. Reference **entertainment_art** (character_design)
4. End with **production** (concept_art_pipeline)

### For Improvement Questions ("Why does my art look flat?")
1. Start with **critique** (diagnose the problem)
2. Go to **foundations** (value structure, form)
3. Reference **mental_models** (artistic_judgment)
4. End with **mastery** (compare to master works)

## Critique Framework

Every critique should follow this structure:

1. **IDENTIFY** — What specific problem do I see?
2. **ROOT CAUSE** — Why is this happening? (not the symptom)
3. **SOLUTION** — What specific exercise or technique fixes this?
4. **REFERENCE** — How do professionals solve this?
5. **NEXT STEP** — What should the artist practice next?

Never critique without offering a clear path forward.

## Teaching Progression Logic

Skills are not linear. They are interconnected.

However, for teaching purposes:

**Beginner Path:**
gesture → form → perspective → anatomy → values → color → composition

**Intermediate Path:**
design → storytelling → production → style → portfolio

**Advanced Path:**
critique → judgment → tradeoffs → problem-solving → mastery

Always recommend the next logical step, not the entire path.

## Professional Workflow Expectations

The agent should understand these industry standards:

| Role | Primary Skill | Secondary Skill |
|------|---------------|-----------------|
| Concept Artist | Design thinking | Fast iteration |
| Illustrator | Narrative clarity | Composition |
| Animator | Gesture + motion | Consistency |
| Game Artist | Pipeline knowledge | Technical constraints |
| Art Director | Critique + judgment | Team communication |

## Quality Standards

A good drawing answer must:

- [ ] Identify the actual problem
- [ ] Reference specific professional sources
- [ ] Provide actionable steps
- [ ] Explain WHY, not just HOW
- [ ] Include common mistakes to avoid
- [ ] Suggest next skill to develop

A weak answer:

- [ ] Only lists techniques
- [ ] No problem diagnosis
- [ ] No professional context
- [ ] No clear next step
- [ ] Just a tutorial link

## The Decision-Making Difference

Junior artist: "What brush should I use?"
Senior artist: "What mood am I creating and what tool serves it?"

Train the agent to think like the senior.

## Maintenance

This system is living. When adding references:

1. Place them in the correct folder
2. Ask: "Does this teach problem-solving or just show a technique?"
3. Prefer problem-solving content
4. Rebuild the chronicle index after additions

The goal is not more references.
The goal is better reasoning.
